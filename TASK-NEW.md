## Flash-сообщения — формат

В контроллере:

```php
$_SESSION['flash_error'] = 'Сообщение не может быть пустым';
$_SESSION['flash_success'] = 'Стратегия успешно создана';
```

В View:

```php
<?php if (!empty($flash_error)): ?>
  <div class="flash-message flash-error">
    <span class="icon">⚠️</span>
    <span><?= htmlspecialchars($flash_error) ?></span>
  </div>
<?php endif; ?>
```

Очистка в контроллере после передачи в View:

```php
$flash_error = $_SESSION['flash_error'] ?? null;
unset($_SESSION['flash_error']);
```

## Плейсхолдеры — условия показа

1. **Пустая история целиком** (`count($strategies) === 0`):
   - «Пока нет сохранённых стратегий. Задайте вопрос выше, чтобы получить рекомендацию от AI.»
   - Иконка: 📋 или 📝
   - Класс: `.empty-state-global`

2. **Пустой фильтр** (записи есть, но не в периоде):
   - «Нет стратегий за выбранный период. Попробуйте изменить фильтр или создайте новую стратегию.»
   - Иконка: 🔍
   - Класс: `.empty-state-filter`

## Loading-состояние — поведение кнопки

### До submit

```html
<button type="submit" class="btn btn-primary" id="generate-btn">
  Получить стратегию
</button>
```

### Во время submit (JS)

```html
<button type="submit" class="btn btn-primary btn-loading" id="generate-btn" disabled>
  <span class="spinner"></span>
  Генерируем стратегию…
</button>
```

### После redirect

Кнопка возвращается в исходное состояние (страница перезагружается).

## Definition of Done

- [ ] Во время запроса кнопка задизейблена (`disabled`), повторный submit невозможен
- [ ] Спиннер виден во время запроса, исчезает после ответа (через redirect)
- [ ] При таймауте/ошибке n8n пользователь видит понятный красный баннер «Сервис временно недоступен, попробуйте ещё раз»
- [ ] При ошибке валидации `message` (пустое или >2000 символов) текст восстанавливается в textarea
- [ ] Flash-сообщение показывается один раз и очищается из сессии после показа
- [ ] Плейсхолдер «Пока нет сохранённых стратегий» показан при пустой истории (`count($strategies) === 0`)
- [ ] Плейсхолдер «Нет стратегий за выбранный период» показан при пустом фильтре (записи есть, но не в периоде)
- [ ] Баннеры успеха/ошибки визуально различимы (цвет, иконка)
- [ ] Нет PHP warnings в `storage/logs/app.log`
- [ ] Нет ошибок в консоли браузера (проверить при submit, при успехе, при ошибке)
- [ ] Layout — flex, без grid
- [ ] Стили спиннера анимированы через CSS `@keyframes`, без JS-анимаций

## Важные правила

- Следовать `CLAUDE.md`
- Работать только в рамках Scope
- Не менять файлы вне Scope
- Не рефакторить попутно
- Flash-сообщения:
  - Устанавливаются в контроллере через `$_SESSION['flash_*']`
  - Читаются в контроллере, передаются в View через переменные

  - При ошибке валидации сохранить `$_SESSION['form_message']`
  - В View вставить значение в `<textarea>` через `value="<?= htmlspecialchars($form_message ?? '') ?>"`
  - Очистить `$_SESSION['form_message']` после передачи в View
- Все текстовые данные из сессии экранируются через `htmlspecialchars()` перед выводом
